# 용어 정의 (Glossary)

이 문서는 프로젝트 전체에서 사용되는 용어를 정의한다. 모든 문서와 코드에서 동일한 용어를 사용하여 일관성을 유지한다.

---

## 핵심 용어

### 콘텐츠 관련

| 용어 | 영문 | 정의 |
|------|------|------|
| **콘텐츠 타입** | Content Type | 사용자가 정의하는 데이터 구조. 예: 게시글, 상품, FAQ |
| **콘텐츠** | Content | 콘텐츠 타입의 실제 데이터 인스턴스. 예: 개별 게시글, 개별 상품 |
| **필드** | Field | 콘텐츠 타입을 구성하는 개별 속성. 예: 제목, 내용, 가격 |
| **필드 타입** | Field Type | 필드의 데이터 유형. 예: 텍스트, 숫자, 이미지, 관계 |
| **관계** | Relation | 콘텐츠 타입 간의 연결. 예: 게시글 - 작성자 (1:N) |

### 페이지 관련

| 용어 | 영문 | 정의 |
|------|------|------|
| **페이지** | Page | 사용자에게 보여지는 화면 단위 |
| **페이지 빌더** | Page Builder | 드래그앤드롭으로 페이지를 구성하는 도구 |
| **컴포넌트** | Component | 페이지를 구성하는 재사용 가능한 UI 블록 |
| **템플릿** | Template | 페이지 생성 시 사용하는 미리 정의된 레이아웃 |
| **레이아웃** | Layout | 페이지의 전체 구조 (헤더, 푸터, 사이드바 등) |
| **위젯** | Widget | 특정 기능을 수행하는 작은 컴포넌트. 예: 검색창, 최근 게시물 |

### 페이지 타입

| 용어 | 영문 | 정의 |
|------|------|------|
| **메인페이지** | Main Page | 사이트 홈 페이지 (사이트당 1개) |
| **일반 페이지** | Static Page | 정적 콘텐츠 페이지. 예: 회사소개, 연락처 |
| **동적 페이지** | Dynamic Page | 콘텐츠와 연동되는 페이지. 예: 게시판, 상품 목록 |
| **시스템 페이지** | System Page | 필수 기능 페이지. 예: 로그인, 404, 회원가입 |

### 사용자 관련

| 용어 | 영문 | 정의 |
|------|------|------|
| **회원** | Member | Public Site를 이용하는 일반 사용자 |
| **관리자** | Admin | Admin Panel에 접근할 수 있는 사용자 |
| **역할** | Role | 권한의 집합. 예: 슈퍼관리자, 편집자, 뷰어 |
| **권한** | Permission | 특정 작업을 수행할 수 있는 자격 |

### 플랫폼 관련

| 용어 | 영문 | 정의 |
|------|------|------|
| **Admin Panel** | Admin Panel | 관리자용 내부 페이지 (콘텐츠/시스템 관리) |
| **Public Site** | Public Site | 방문자용 외부 공개 웹사이트 |
| **API** | API | 외부에서 콘텐츠에 접근하기 위한 인터페이스 |

### 워크플로우 관련

| 용어 | 영문 | 정의 |
|------|------|------|
| **초안** | Draft | 작성 중인 콘텐츠 (미발행) |
| **검토** | Review | 승인 대기 중인 상태 |
| **승인** | Approved | 승인 완료된 상태 |
| **발행** | Published | 공개된 상태 |
| **반려** | Rejected | 승인 거부된 상태 |
| **예약 발행** | Scheduled | 특정 시간에 자동 발행 예정 |

### 기타

| 용어 | 영문 | 정의 |
|------|------|------|
| **미디어** | Media | 이미지, 파일, 동영상 등 업로드된 자원 |
| **슬러그** | Slug | URL에 사용되는 식별자. 예: `/post/hello-world` |
| **버전** | Version | 콘텐츠의 수정 이력 |
| **Webhook** | Webhook | 이벤트 발생 시 외부 URL로 자동 알림 |
| **i18n** | i18n | 다국어 지원 (Internationalization) |

## 약어

| 약어 | 전체 | 설명 |
|------|------|------|
| CRUD | Create, Read, Update, Delete | 기본 데이터 조작 |
| SEO | Search Engine Optimization | 검색 엔진 최적화 |
| OG | Open Graph | SNS 공유 메타 태그 |
| SSR | Server-Side Rendering | 서버 측 렌더링 |
| CSR | Client-Side Rendering | 클라이언트 측 렌더링 |
| JWT | JSON Web Token | 인증 토큰 |

---

## 용어 사용 규칙

1. **한글 우선**: 문서에서는 한글 용어를 우선 사용
2. **코드는 영문**: 변수명, 함수명 등 코드에서는 영문 용어 사용
3. **일관성 유지**: 동일한 개념에 대해 항상 같은 용어 사용
   - (O) 콘텐츠 타입
   - (X) 컨텐츠 타입, 콘텐트 타입, 콘텐츠 유형

## 혼동하기 쉬운 용어

| 구분 | 용어 A | 용어 B | 차이점 |
|------|--------|--------|--------|
| 사용자 | 회원 (Member) | 관리자 (Admin) | 회원은 Public Site 이용자, 관리자는 Admin Panel 이용자 |
| 페이지 | 페이지 (Page) | 템플릿 (Template) | 페이지는 실제 화면, 템플릿은 페이지의 기본 틀 |
| 데이터 | 콘텐츠 타입 | 콘텐츠 | 콘텐츠 타입은 구조 정의, 콘텐츠는 실제 데이터 |
| UI | 컴포넌트 | 위젯 | 컴포넌트는 범용 UI 블록, 위젯은 특정 기능 컴포넌트 |
